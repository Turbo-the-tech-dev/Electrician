This curriculum design establishes a rigorous, safety-focused training program for Industrial Automation and Control Systems (IACS). This discipline is the logical progression from general electrical installation, bridging the gap between standard wiring and the intelligent systems that drive modern manufacturing.
Program Title: Certified Industrial Automation Technician (CIAT)
Total Contact Hours: 180 Hours (90 Theory / 90 Lab)
Prerequisite: Professional Electrical Installation Certification or equivalent experience.
1. Core Curriculum Structure
The program is divided into 9 modules, moving from component-level understanding to system-level integration.
| Module | Title | Key Concepts (Theory) | Lab Competencies (Hands-on) |
|---|---|---|---|
| 01 | Safety & Standards in Automation | NFPA 70E (Arc Flash), NFPA 79 (Industrial Machinery), Risk Assessment, OSHA 1910. | LOTO verification on energized panels; PPE selection drill; Emergency Stop circuit validation. |
| 02 | Control Circuit Fundamentals | Relays, contactors, solenoids, logic symbols (N.O./N.C.), holding circuits, interlocks. | Wiring a reversing motor starter with interlocks; Building a relay logic timing circuit.  |
| 03 | Sensors & Instrumentation | Discrete vs. Analog, Proximity (Inductive/Capacitive), Photoeyes, 4-20mA loops, Thermocouples. | Calibrating a pressure transmitter; Wiring NPN vs. PNP sensors; Troubleshooting loop breaks. |
| 04 | Variable Frequency Drives (VFDs) | PWM theory, V/Hz control, harmonics, braking resistors, parameter sets. | Programming VFD acceleration/deceleration; Configuring digital inputs for multi-speed control. |
| 05 | PLC Hardware & Architecture | CPU, I/O modules, Sinking/Sourcing, Scan cycle, Memory organization, Addressing. | Racking hardware; I/O checkout procedure; Wiring a PLC input card. |
| 06 | PLC Programming Level I | Ladder Logic standards (IEC 61131-3), Timers (TON/TOF), Counters, Math instructions. | Writing a "Traffic Light" sequence; Programming a conveyor start/stop station. |
| 07 | PLC Programming Level II | Subroutines, Analog scaling, PID loops, Structured Text basics, Fault routines. | Programming a tank level control loop; Creating a specialized function block. |
| 08 | HMI & SCADA Design | Tag databases, UI/UX best practices for operators, Alarming hierarchy, Trending. | Designing a touch-screen operator panel; Mapping PLC tags to HMI buttons/indicators. |
| 09 | Industrial Networks & Cybersecurity | Ethernet/IP, Modbus, PROFINET, Switch management, Defense-in-Depth security. | Assigning static IPs; Setting up a Managed Switch; Isolating a control network VLAN. |
2. Detailed Content Outline: Sample Chapter (Module 05)
To demonstrate the pedagogical rigor, here is the breakdown for Module 05: PLC Hardware & Architecture.
A. Theory Component (4 Hours)
 * The Scan Cycle: Input Scan \rightarrow Program Execution \rightarrow Output Scan.
 * I/O Types: Difference between Discrete (24VDC/120VAC) and Analog (Voltage/Current).
 * Sinking vs. Sourcing: The physics of current flow in input cards.
 * Safety Callout: Never force inputs/outputs on running machinery without visual verification.
B. Lab Component (4 Hours)
 * Activity 5.1: Hardware Assembly. Student mounts power supply, CPU, and I/O cards to DIN rail.
 * Activity 5.2: Wiring "The Common Mistake." Student intentionally wires a sinking sensor to a sinking card (on a mock bench) to observe failure, then corrects it to sourcing.
 * Activity 5.3: The "Smoke Test" Protocol. Step-by-step power-up procedure to verify voltage levels before engaging the CPU.
C. Assessment
 * Quiz: 10 questions (e.g., "Calculate the current draw for a rack of 4 modules").
 * Skill Sign-off: Instructor verifies the student can replace a fused output card in under 5 minutes while maintaining safety protocols.

4. Assessment Strategy
Comprehensive Final Exam (Capstone)
 * Format: 50 Questions + 1 Practical Challenge.
 * Question Mix:
   * 30% Theory (Physics/Logic).
   * 30% Code & Safety (NEC/NFPA).
   * 40% Scenario-based (Troubleshooting).
 * The Practical Challenge: "The Black Box."
   * Students are given a system with 3 introduced faults (software bug, blown fuse, broken wire).
   * Goal: Return system to full operation in < 45 minutes.
Quality Assurance Checklist
 * Are all circuit diagrams compliant with current IEEE/ANSI symbols?
 * Do all HMI screenshots have Alt-Text for accessibility?
 * Are safety warnings (DANGER, WARNING, CAUTION) color-coded to ANSI Z535 standards?
